# 🏠 赛博室友 — 项目讲解脚本

---

## ⏱ 总时长：5-6 分钟

| 环节 | 时长 | 内容 |
|------|------|------|
| 开场 hook | 30s | 抛出问题，引起共鸣 |
| 项目简介 | 40s | 一句话说清楚是什么 |
| 技术架构 | 60s | 多智能体 + 记忆 + 人格 |
| 现场演示 | 2min | 边点边讲，最核心的环节 |
| 创新亮点 | 40s | 为什么比别人牛 |
| 总结收尾 | 30s | 一句话记住你 |

---

## 🎤 第一部分：开场 Hook（30秒）

> **"在座的各位，都住过宿舍吧？"**
>
> **"有没有想过——如果把四个不同 MBTI 的人关进一个 AI 宿舍，让他们自己聊天、吵架、和好，会发生什么？"**
>
> **"我今天带来的项目，就是这样一个实验——「赛博室友」，一个多智能体宿舍模拟器。"**

---

## 🎤 第二部分：项目简介（40秒）

> **"简单来说，我做了四只有人格的 AI，把它们丢进一间虚拟宿舍。"**
>
> - 🐶 **小明，ENFP 社牛**——负责带节奏、跑题、搞气氛
> - 🦉 **阿哲，INTJ 卷王**——负责毒舌、理性分析、怼人
> - 🌿 **小宇，ISFP 养生人**——负责 peace、emo、劝大家早睡
> - 🐒 **老王，ESTP 行动派**——负责吃、搞事情、抄作业
>
> **"它们不是简单的聊天机器人——每个 Agent 有自己的记忆、情绪、甚至对其他人的好感度。它们真的会记仇，也会感动。"**

---

## 🎤 第三部分：技术架构（60秒）

> **"来看一下技术架构。我画了张图——"**

```
（指屏幕上的架构图或直接画）

┌─────────────────────────────────┐
│      Streamlit 前端界面          │  ← 用户围观 & 附身发言
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│       模拟引擎 (Simulator)        │  ← 调度谁说话、注入事件
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│   Agent×4      │  记忆系统       │
│  ·人格 Prompt  │  ·短期记忆(20条) │
│  ·情绪状态     │  ·长期记忆(重要) │
│  ·回复生成     │  ·好感度矩阵     │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│     DeepSeek API (大模型)        │  ← 驱动每个 Agent 的"大脑"
└─────────────────────────────────┘
```

> **"三个核心技术点："**
>
> **"第一，每个 Agent 是独立的——它有自己的 System Prompt 定义人格，有自己的记忆空间。小明说的话，阿哲会从自己的视角去理解，而不是共享一个大脑。这就是「多智能体」和「单聊天机器人」的本质区别。"**
>
> **"第二，双层记忆系统。短期记忆存最近 20 条对话，保证上下文连贯；长期记忆只存重要事件——比如吵架、感动瞬间——用关键词匹配自动筛选。不需要向量数据库也能跑。"**
>
> **"第三，好感度动态变化。每个 Agent 回复里可以带隐藏标记，比如 '老王说得对 <!--rel:老王:+1-->'，系统解析后自动更新关系分。所以你会看到侧边栏的好感度条是真的在变。"**

---

## 🎤 第四部分：现场演示（2分钟）⭐ 核心

> **"废话不多说，直接看效果。"**

---

### 演示步骤 1：初始状态（10秒）

> *打开浏览器，展示主界面*
>
> **"这是 302 宿舍直播间。左边四个室友卡片，每个人有 MBTI 标签、当前心情、以及对其他三人的好感度。初始都是 50 分，普通室友关系。"**
>
> *鼠标在侧边栏滑动展示*

---

### 演示步骤 2：第一轮对话（20秒）

> *点击「💬 下一轮对话」*
>
> **"我点一下「下一轮对话」，系统自动选一个室友发言——"**
>
> *等待回复出现*
>
> **"注意看，它不是简单的一问一答。每个 Agent 说话时，其他三个人都在'听'——这些内容会进入他们的短期记忆，影响他们后面说什么。"**

---

### 演示步骤 3：连续多轮（20秒）

> *再点 3-4 轮，展示对话自然流动*
>
> **"我再推进几轮。你会发现对话是自然流动的——不会出现机器人感。因为每个人的 Prompt 里写死了口语风格、网络梗、口头禅。"**
>
> **"而且——注意到没有——它们会互怼。阿哲经常怼小明，老王经常打圆场，小宇经常 '我都可以呀～'。这不是我写的剧本，是 LLM 根据人格自己发挥的。"**

---

### 演示步骤 4：随机事件（25秒）⭐ 最有展示效果

> *点击「🎲 触发随机事件」*
>
> **"接下来是最有意思的功能——随机事件注入。"**
>
> *事件出现，下一个 Agent 对此做出反应*
>
> **"比如这个事件：'期末考试成绩出来了，有人欢喜有人愁。' 这不是我预设的对话，是系统从事先准备的 30+ 个宿舍事件里随机抽取，然后所有 Agent 同时感知到这个事件，下一个发言的人会自然地对此做出反应。"**
>
> **"你可以再点一次，每次事件都不一样——外卖到了、停电了、有人失恋了——这就是模拟真实宿舍的不可预测性。"**

---

### 演示步骤 5：附身模式（25秒）

> *在侧边栏「附身发言」选择小明，输入："兄弟们，我好像喜欢上隔壁班的女生了"*
>
> **"最后一个功能——附身模式。假设我想搞点事情，我选择附身小明，替他说话。"**
>
> *点击发送，观察其他人的反应*
>
> **"你看，阿哲开始理性分析了，老王开始出馊主意了，小宇开始 emo 了。这就是多智能体的魅力——一个输入，四个不同人格的视角。"**
>
> **"而且附身的这句话也会进入所有 Agent 的记忆，后续对话会一直受它影响。"**

---

### 演示步骤 6：好感度变化（15秒）

> *切回侧边栏，指出好感度变化*
>
> **"最后注意看好感度——经过这些对话后，关系已经不是初始的 50 了。有人关系变铁了，有人可能被怼得掉分了。这些变化是对话中自然产生的，不是硬编码的。"**

---

## 🎤 第五部分：创新亮点（40秒）

> **"总结一下，为什么这个项目不一样："**
>
> **"第一，多智能体而非单机器人。市面上 99% 的 AI 聊天产品都是你和 AI 一对一聊。我这里是一群 AI 互相聊，你是上帝视角围观——这是一个微缩版的社交实验。"**
>
> **"第二，真正的记忆和关系系统。每个 Agent 有独立记忆空间，好感度随对话动态演化。这不是 feature，这是 emergent behavior——涌现行为。"**
>
> **"第三，场景即共鸣。我没有做什么高大上的企业级场景，选的就是每个人每天经历的宿舍生活。但正是这个选择让评审的人能一秒代入，不需要解释什么是 MBTI，什么是外卖拼单。"**

---

## 🎤 第六部分：总结（20秒）

> **"最后用一句话总结这个项目——"**
>
> **"「赛博室友」不只是四个聊天机器人，它是一个微缩的社会模拟器。你改变一个变量——注入一个事件、附身说一句话——然后观察四个不同人格的 AI 如何反应、如何传播、如何形成集体记忆。这就是多智能体最迷人的地方。"**
>
> **"谢谢大家，欢迎提问！"**

## 💡 Q&A 备答

| 可能的问题 | 准备的回答 |
|-----------|-----------|
| **"用的什么大模型？"** | "DeepSeek，国产模型，API 兼容 OpenAI SDK。架构设计成模型无关的，换成 GPT 或者 Claude 只需要改两行配置。" |
| **"记忆系统怎么实现的？"** | "短期记忆是环形缓冲区，长期记忆用关键词重要性打分。MVP 不需要向量数据库，但架构预留了扩展接口。" |
| **"好感度变化是 LLM 自己判断的吗？"** | "对，Prompt 里约定了一个隐藏标记格式 `<!--rel:名字:±1-->`，LLM 自己判断该加还是该减，后端解析更新。比用情感分析模型更自然。" |
| **"能做到多少人？"** | "目前 4 人宿舍刚好——人太少没互动，人太多对话会发散。架构上可以扩展到 N 个 Agent，但 Token 消耗和对话质量需要平衡。" |
| **"延迟多久？"** | "一轮对话大概 2-4 秒，主要是 LLM API 耗时。批量跑 5 轮大概 10-15 秒。" |

## 🎬 演示技巧

1. **不要照着念**——脚本是骨架，用自己的话讲
2. **演示优先**——图片里结构图可以提前画好（PPT 或 excalidraw），边指边讲
3. **随机事件是最佳亮点**——如果时间紧，优先展示这个
4. **提前跑几轮**——演示前先点 3-4 轮，让对话有上下文，展示时"下一轮"的回复会更自然
5. **如果某个回复崩了**——直接说"这就是 LLM 的随机性，就像真实室友有时候也说奇怪的话"，然后点下一轮
6. **录屏备选**——如果网络不稳，提前录一段 2 分钟的操作视频
